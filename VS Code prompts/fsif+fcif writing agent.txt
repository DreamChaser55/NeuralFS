Read the @/<campaign_folder>/Campaign_Bible.txt . Read the mission design docs in @/<campaign_folder>/Detailed_Mission_Design_Documents folder.

1. Inspect the fiction viewer text files in the Detailed_Mission_Design_Documents folder (the files named according to the schema "missionname_story.txt"). Add text styling tags to these fiction viewer text files (where appropriate), if they don't already contain them. After editing, validate each fiction viewer file with the Fiction Viewer Validator (`fiction_viewer_validator.py`). Fix any reported errors or warnings before proceeding.
2. Write fsif missions based on the provided campaign Bible and mission design docs. Place them into the '/<campaign_folder>/fsif/' folder. When writing FSIF files, explicitly define the `audio.tts_provider` field (valid options: `"google"`, `"elevenlabs"`, `"inworld"`, or `"none"`). Match the voices to your chosen provider, or use `"none"` if the mission has no voiced lines. For each mission you write, immediately use the FSIF to FS2 Converter (`fsif_to_fs2.py`) to convert it to `.fs2`. If the Converter reports any errors or warnings during conversion, analyze the problematic file, fix the issues, and iterate until the file converts cleanly. Once a mission cleanly converts to `.fs2`, proceed to write the next fsif mission.
3. After all fsif missions are written and cleanly converted, write a FCIF campaign definition file based on the campaign Bible, mission design docs and mission progression in the written fsif files. Place it into the '/<campaign_folder>/' folder (alongside Campaign_Bible.txt). Immediately use the FCIF to FC2 Converter (`fcif_to_fc2.py`) to convert the `.fcif` file to `.fc2`. The converter will automatically check the `.fsif` files to verify that all player starting wings ships and weapons referenced in the missions are properly unlocked before use. If the Converter reports any errors or warnings, fix the file(s) and iterate until the `.fcif` cleanly converts.
   IMPORTANT — first mission loadout completeness: Every ship class and every weapon (primary and secondary) that appears in the first FSIF mission in player starting wings **must** be listed in `starting_loadout`. There is no prior mission in which `allow-ship` or `allow-weapon` SEXPs could have run, so anything absent from `starting_loadout` will simply not appear in the first mission in-game. After writing the FCIF, cross-check it against the first mission's FSIF to verify this. All ships and weapons that are introduced in subsequent missions must be allowed with `allow-ship` or `allow-weapon` SEXPs.

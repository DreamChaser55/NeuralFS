  Backgrounds and Nebulae

*mission-set-nebula*
    mission-set-nebula
    Turns nebula on/off
    Takes 1 or 2 arguments...
    1: 0 for nebula off, 1 for nebula on
    2: Nebula range (optional; defaults to 3000)
*mission-set-subspace*
    mission-set-subspace
    Turns subspace on/off
    Takes 1 argument...
    1: 0 for subspace off, 1 for subspace on
*change-background*
    change-background
    Sets the displayed suns and bitmaps to one of the stored
    mission backgrounds. Takes 1 argument...
    1: Background number (starting from 1)
*add-background-bitmap*
    add-background-bitmap (deprecated)
    Adds a background bitmap to the sky. Returns an integer that
    is stored in a variable so it can be deleted using remove-
    background-bitmap. NOTE: This version of the SEXP uses the
    old incorrectly-calculated angle math. Use add-background-
    bitmap-new instead.

    Takes 8 to 9 arguments...
    1: Background bitmap name
    2: Pitch
    3: Bank
    4: Heading
    5: X scale (expressed as a percentage of the original size
    of the bitmap)
    6: Y scale (expressed as a percentage of the original size
    of the bitmap)
    7: X divisions.
    8: Y divisions.
    9: Numeric variable in which to store the result (optional)
*add-background-bitmap-new*
    add-background-bitmap-new

    1: Background bitmap name
    2: Pitch
    3: Bank
    4: Heading
    5: X scale (expressed as a percentage of the original size
    of the bitmap)
    6: Y scale (expressed as a percentage of the original size
    of the bitmap)
    7: X divisions.
    8: Y divisions.
    9: Numeric variable in which to store the result (optional)
*remove-background-bitmap*
    remove-background-bitmap
    Removes the nth background bitmap from the mission

    Takes 1 argument...
    1: Zero based bitmap index from the 'Bitmap' box in the
    background editor
    You can also use the result of a previous call to add-
    background-bitmap to remove that added bitmap

    Takes 1 argument...
    1: Will be shown
*set-skybox-model*
    set-skybox-model
    Sets the current skybox model. Takes 1-7 arguments
    1: Model filename (with .pof extension) to switch to
    2: Restart animated textures, if they exist (optional,
    defaults to true)
    3-8: Set or unset the following skyboxes flags (optional)
    add-lighting, no-transparency, add-zbuffer
    add-culling, no-glowmaps, force-clamp

    Note: If the model filename is set to "default" with no
    extension then it will switch to the mission supplied
    default skybox. If it is an empty string or "none" then it
    will remove any current skybox.
*set-skybox-orientation*

*set-skybox-alpha*

    3: Blue (0 - 255).
*toggle-asteroid-field*
    toggle-asteroid-field
    Turns an existing asteroid or debris field on/off. Will do
    nothing if no field is defined in the mission file or set
    with sexps.
    Note that this differs from actually removing and adding a
    field in that it only toggles a field from being rendered or
    calculated.
    Takes 1 argument...
    1: true for field on, false for field off
*set-asteroid-field*
    set-asteroid-field (deprecated in favor of config-asteroid-
    field)
    Creates or overwrites the asteroid or debris field with an
    asteroid field.
    Takes 1 or more arguments...
    1: Number of asteroids in the field, or 0 to remove an
    existing field
    2: 0 for passive field, 1 for active field, 2 for enhanced
    passive, 3 for enhanced active, defaults to 0
    3: The speed of the asteroids, defaults to 0
    4: True to enable brown asteroids, defaults to true
    5: True to enable blue asteroids, defaults to false
    6: True to enable orange asteroids, defaults to false
    7: Outerbox Min X, defaults to -1000
    8: Outerbox Max X, defaults to 1000
    9: Outerbox Min Y, defaults to -1000
    10: Outerbox Max Y, defaults to 1000
    11: Outerbox Min Z, defaults to -1000
    12: Outerbox Max Z, defaults to 1000
    13: True to activate the inner box where no asteroids will
    be spawned, defaults to false
    14: Innerbox Min X, defaults to -500
    15: Innerbox Max X, defaults to 500
    16: Innerbox Min Y, defaults to -500
    17: Innerbox Max Y, defaults to 500
    18: Innerbox Min Z, defaults to -500
    19: Innerbox Max Z, defaults to 500
    Rest: The ships the asteroid field will target if it's an
    active field

*config-field-targets*
    config-field-targets
    Adds or removes ships from the Asteroid Field targets list.
    Only works on Active fields.
    You can remove all targets at once by supplying 0 ship
    arguments.
    Removing all targets will force the field to use default
    behavior and target the first ship on the escort list.
    The inverse is not true. Ship arguments must be provided to
    add targets.
    Takes 1 or more arguments...
    1: True to add to the list, false to remove
    Rest: The names of ships to add or remove
